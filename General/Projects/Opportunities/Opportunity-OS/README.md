# Opportunity-OS — Brian Kipkirui

**A personal knowledge base and execution system for finding, evaluating, and applying to PhD positions, fellowships, and jobs.**

---

## How to Use This Vault

### First time setup
1. Open this folder as an Obsidian vault
2. Install the Dataview plugin (for table queries in Home.md)
3. Read `01_Profile/Profile_Summary.md` — this is the master context
4. Read `01_Profile/Positioning.md` — this is the concise narrative frame
5. Update all `[UPDATE]` fields throughout the profile files
6. Open `00_Home/Home.md` as your starting point every session

### Weekly workflow
1. Open `00_Home/Now.md` — update priorities
2. Run the Scout pipeline → new opportunities in `04_Opportunities/Inbox/`
3. Run the Evaluator → scored and filed opportunities
4. Review `00_Home/Weekly_Application_Queue.md`
5. Run Drafter for confirmed applies

### Working with AI agents
All agent instructions are in `06_Agents/`. When running an agent:
- Always provide `01_Profile/Profile_Summary.md` as context
- Provide the specific opportunity note for drafting tasks
- Provide `05_Appraisals/Scoring_Rubric.md` for evaluation tasks

## Folder Map

```
00_Home/          → Dashboard, weekly queue, current focus
01_Profile/       → Stable facts about Brian (interests, skills, fit, immigration...)
02_Assets/        → Reusable documents (CVs, templates, paragraph modules)
03_Portfolio/     → One file per project with multi-framing descriptions
04_Opportunities/ → One file per opportunity; subfolders by type
05_Appraisals/    → Scoring rubric, decision rules, comparison table
06_Agents/        → Instructions for Scout, Evaluator, Drafter, Tracker agents
07_Pipelines/     → Step-by-step SOPs for each workflow
08_Reference/     → Target labs, country visa notes, application history
09_Archive/       → Closed, rejected, or skipped opportunities
```
