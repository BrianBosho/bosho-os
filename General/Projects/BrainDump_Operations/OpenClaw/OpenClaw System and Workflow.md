# OpenClaw System and Workflow

Source: [[../../00 Inbox/Workbook q2 wk1|Workbook q2 wk1]]

## System needs expressed in the source doc

- Reorganize Notion and Drive so the agent stack works against cleaner source material.
- Allow agents to read email.
- Add bookmark ingestion through the Raindrop API.
- Build a Trello-like dashboard or equivalent execution view.
- Create a reusable workflow for organizing projects, ideas, and knowledge in Obsidian.

## Opportunities-specific OpenClaw work

- Build cron jobs to populate opportunity pages and databases.
- Support a daily or every-two-days review routine.
- Help compile requirements and application checklists.
- Draft application documents from an existing knowledge base.
- Convert opportunity work into to-do lists and deadline-aware execution.

## Review content embedded in the doc

The workbook also contains an OpenClaw review noting:

- config drift from the intended multi-model plan
- subagent failures around model switching
- overly broad exec approvals
- incomplete POS agent setup
- exposed secrets in config

## Practical interpretation

- The source doc treats OpenClaw as an execution engine, not the source of truth.
- The durable source-of-truth materials should live in Drive, Notion, and Obsidian with a cleaner structure.
