# Weekly Planning System

Source: [[../00 Inbox/Workbook q2 wk1|Workbook q2 wk1]]

## Weekly priorities captured from the source doc

- Move FedProp toward submission and update Joao on progress by the end of the week.
- Choose a concrete VLM direction and define the paper structure.
- Review master's-student research projects and discuss them with Joao.
- Prepare the KTH PhD application.
- Submit or prepare at least one fellowship application.
- Apply for at least one job.
- Review EML research and build a project page/resources list.
- Spend time on mechanistic-interpretability resources.
- Check Australian scholarships and TU Munich positions.

## Long-term commitments named in the source doc

- Become a stronger researcher and improve the research process.
- Expand outward through PhD, collaboration, job, and fellowship opportunities.
- Develop competence through deliberate reading and study.
- Improve concentration and clarity.

## Scheduling model proposed in the source doc

- Three 3-hour blocks per day, tentatively:
  - 9am-12pm
  - 2pm-5pm
  - 8pm-11pm
- Saturday can carry extra completion time.
- Sunday is mainly for planning.

## Task design constraints

- Use two back-to-back 90-minute work blocks within a 3-hour slot.
- A 90-minute slot can hold one task or a split such as 45/45 or 60/30.
- No task in the database should take more than 90 minutes.
- Every task should have an explicit outcome.

## Work categories

- Research work
- Personal work: side projects, PhD applications, software work
- Admin and miscellaneous obligations

## Why this matters

The source doc repeatedly points to the same failure mode: too many parallel commitments without a bounded execution structure. This planning note captures the execution model that the rest of the BrainDump should align to.

## Weekly planning checklist

- [ ] Pick the 3-5 highest-value outcomes for the week.
- [ ] Translate each outcome into tasks no larger than 90 minutes.
- [ ] Assign each task to a work category.
- [ ] Place tasks into the weekly time blocks.
- [ ] Reserve overflow or catch-up time.
- [ ] Keep Sunday mainly for planning and review.
- [ ] Review progress daily and re-plan when blocks slip.

## Weekly execution board

| Workstream | This week | Concrete outcome |
|---|---|---|
| FedProp | move toward submission and update Joao | draft or experiment progress plus update sent |
| VLM grounding | choose direction and define paper structure | one clear thesis and outline |
| Research mentoring | review master's-student projects | discussion-ready notes |
| Applications | prepare KTH and at least one fellowship or job application | one completed application package |
| Research development | review EML and mechanistic-interpretability resources | resource list or notes page |
| Target search | review Australian scholarships and TU Munich positions | updated shortlist |

## Block template

| Block | Default use | Rule |
|---|---|---|
| 9am-12pm | hardest deep work | one or two bounded tasks |
| 2pm-5pm | second major work block | execution or follow-through |
| 8pm-11pm | lighter but still intentional work | reading, applications, cleanup |
| Saturday extra time | overflow or completion | finish anything critical |
| Sunday | planning | review and re-scope |

## Task sizing rules

| Rule | Meaning |
|---|---|
| Max 90 minutes | no task should be vague or too large |
| Explicit outcome | every task ends with a visible artifact |
| Category tag | every task belongs to research, personal, or admin |
| Replan quickly | slipping tasks must be rescheduled, split, or dropped |
