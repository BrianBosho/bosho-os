# Yes/No Boundary Filter Protocol

## Purpose
To protect prime cognition, enforce work boundaries, and prevent psychological burnout. This protocol acts as a filter for all new requests, tasks, and project extensions during Summer 2026.

---

## Core Boundary Rules

1. **The Bounded Scope Rule:** 
   * **Say NO to:** Any task that does not have a defined, documented scope.
   * **Say YES to:** Bounded work blocks where the exact stopping criteria are clear before starting.

2. **The Next-Slot Rule:**
   * **Say NO to:** Stopping a work block with unfinished tasks left in a vague state.
   * **Say YES to:** Bounding the end of work by explicitly scheduling the next slot for any unfinished task.

---
