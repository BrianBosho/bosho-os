# BrainDump Dashboard

This page is the single organizing view for the `BrainDump` folder.

Operational control page:
- [[06 Planning/Weekly Action Board|Weekly Action Board]]

It answers four questions:

1. Which notes are raw source or reference material?
2. Which notes are actionable checklists or systems?
3. Which notes are project briefs or idea notes?
4. Which notes are reflections, reviews, or problem statements?

## Recommended structure

| Bucket | What belongs here | Main use |

| Reflections and reviews | quarter themes, problem statements, audits, technical review notes | clarify thinking |

## Recommended one-page operating model

Use the BrainDump as a four-layer stack:

| Layer | Purpose | Notes |
|---|---|---|
| Source | keep the original material safe | workbook source, raw archive, reconciliation |
| Meaning | preserve reflection, diagnosis, and strategic thinking | execution problem, Q2 reflection, review notes |
| Direction | define what each active stream is trying to do | VLM direction, FedProp push, College App brief |
| Execution | convert direction into checklists and weekly blocks | opportunities pipeline, OpenClaw workflow, weekly planning |

The weekly control surface for that execution layer is:

- [[06 Planning/Weekly Action Board|Weekly Action Board]]
