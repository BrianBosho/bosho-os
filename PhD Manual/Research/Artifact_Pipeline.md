# Artifact Pipeline

## Core Rule

Progress should be made visible through artifacts, not only through thought or effort.

## Preferred Artifact Types

- short memos
- research abstracts
- problem statements
- metric catalogs
- experiment plans
- figures and tables
- prototype implementations
- ablation results
- draft write-ups

## Monthly Research Shape

The extracted corpus suggests a repeatable progression:

1. Frame the problem clearly.
2. Survey relevant literature and competing methods.
3. Implement minimal candidate baselines.
4. Run controlled experiments.
5. Compare results and identify the promising line.
6. Build the improved metric, method, or evaluation.
7. Scale the evaluation.
8. Draft the narrative while results are still fresh.

## Weekly Deliverable Bias

Each week should leave behind something that another person can react to:

- a page
- a figure
- a result table
- a code stub
- an experiment memo
- a benchmark comparison

## Why This Matters

Artifacts reduce self-deception, enable collaboration, and convert vague ambition into cumulative work.
