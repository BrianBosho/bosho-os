# Project Pipeline

## Purpose

Prevent diffusion: many interesting ideas, no deliverable.

Every research idea must live somewhere in this pipeline.

## Stages

### 1. Spark

Interesting but vague.

Requires:

- title
- why interesting
- one possible question
- next compression step

### 2. Question

Can be stated as a research question.

Requires:

- one-sentence research question
- why it matters
- 3 closest papers

### 3. Candidate

Possible project.

Requires:

- related work
- model/data/tools
- possible method
- evaluation plan
- feedback source

### 4. Active Project

Current research execution.

Requires:

- weekly tasks
- next experiment
- next artifact
- feedback rhythm
- project brief

### 5. Paper Project

Becoming a submission.

Requires:

- claim
- experiment table
- draft outline
- target venue/workshop
- writing schedule

### 6. Thesis Component

Contributes to dissertation arc.

Requires:

- relationship to thesis question
- durable contribution
- committee/supervisor alignment

## Allocation rule

At any time:

- maximum 1 active project
- maximum 2 candidate projects
- unlimited sparks, but sparks cannot consume serious weekly energy

Default allocation:

- 70% active project
- 20% candidate development
- 10% sparks/exploration

> What artifact will exist by Friday?

If no artifact is coming, activate [[02_Projects/Anti-Diffusion Protocol]].
